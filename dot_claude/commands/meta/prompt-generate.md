# /prompt-generate - Generate Structured Prompts

Generate well-structured prompts for various tasks.

## Usage

```
/prompt-generate <task-type>
/prompt-generate --template <name>
/prompt-generate --from-context
```

## Arguments

- `task-type`: Type of prompt (code, review, debug, explain, etc.)
- `--template <name>`: Use specific template
- `--from-context`: Generate based on current context
- `--output <file>`: Save to file

## Prompt Types

### Code Generation

```markdown
## Task

[What to build]

## Context

- Language: [Python/TypeScript/etc.]
- Framework: [If applicable]
- Existing patterns: [Reference existing code]

## Requirements

- [ ] Requirement 1
- [ ] Requirement 2

## Constraints

- Must work with [existing system]
- Should follow [pattern/style]

## Examples

Input: [example input]
Expected output: [example output]
```

### Code Review

```markdown
## Review Request

[What to review]

## Focus Areas

- [ ] Security
- [ ] Performance
- [ ] Maintainability
- [ ] Test coverage

## Context

- Purpose: [What this code does]
- Changed files: [List]
- Related systems: [Dependencies]

## Specific Questions

1. Is [specific concern] handled correctly?
2. Are there edge cases for [scenario]?
```

### Debugging

```markdown
## Problem

[Description of the issue]

## Error Message

\`\`\`
[Paste error/stack trace]
\`\`\`

## Context

- When it occurs: [Trigger conditions]
- Environment: [OS, version, config]
- Recent changes: [What changed before issue]

## Already Tried

- [Attempt 1 and result]
- [Attempt 2 and result]

## Expected Behavior

[What should happen instead]
```

### Explanation

```markdown
## Question

[What you want explained]

## Context

- Current understanding: [What you know]
- Confusion points: [What's unclear]
- Use case: [Why you need this]

## Preferred Format

- [ ] Step-by-step
- [ ] Code examples
- [ ] Diagram description
- [ ] Comparison with [familiar concept]
```

### Refactoring

```markdown
## Refactoring Goal

[What improvement to make]

## Current Code

\`\`\`[language]
[Paste current code]
\`\`\`

## Issues with Current

- [Problem 1]
- [Problem 2]

## Desired Outcome

- [Quality 1]
- [Quality 2]

## Constraints

- Must maintain [behavior/API]
- Should improve [metric]
```

## Workflow

### Step 1: Identify Task Type

| Task                | Template        |
| ------------------- | --------------- |
| Write new code      | Code Generation |
| Check existing code | Code Review     |
| Fix issue           | Debugging       |
| Understand code     | Explanation     |
| Improve code        | Refactoring     |

### Step 2: Gather Context

Collect:

- Relevant code snippets
- Error messages
- Requirements/constraints
- Examples

### Step 3: Fill Template

Use appropriate template and fill:

- Be specific, not vague
- Include relevant context
- State constraints clearly
- Provide examples when helpful

### Step 4: Generate Prompt

```markdown
## Generated Prompt

[Filled template ready to use]

---

_Generated by /prompt-generate_
_Template: [template-name]_
_Context: [source]_
```

## Custom Templates

Create custom templates in `.claude/templates/`:

```markdown
# Template: my-template

## Section 1

[placeholder: description]

## Section 2

[placeholder: details]
```

Use with:

```
/prompt-generate --template my-template
```
